# 調査・実装・記憶のワークフロー

このリポジトリでの市場調査を、3つのツールに役割分担して回す運用メモ。

| 段階 | ツール | 役割 |
|---|---|---|
| 調査 | NotebookLM | 複数ソースを読み込ませて深掘り調査。Claude Codeのトークンを使わずに広く読ませる調査はここで行う |
| 指示 | Claude Code | 調査結果をもとにコード変更・分析ロジックの実装を指示する |
| 記憶 | Obsidian | 調査結果・実装判断を恒久ノートとして蓄積し、タグ・リンクで関連付ける |

## 1. 調査（NotebookLM）

https://notebooklm.google.com にソースをアップロードして質問する。

このリポジトリで調査ソースになりうるもの:

- `data/dashboard.db` から出力したX投稿・市場データ（CSVエクスポート等）
- `docs/index.html` に出力されている分析コメント
- 関連ニュース記事・決算資料のURL

NotebookLMは無料でソースを大量に読み込ませて質問できるため、
「まず広く調べる」工程をここに寄せることで、Claude Codeのトークン消費を実装作業に集中させる。

## 2. 指示（Claude Code）

NotebookLMでの調査結果・自分の判断をもとに、Claude Codeへ実装を指示する。
例: 新しい指標の収集追加、アラート条件の変更、ダッシュボードの表示改善など。

## 3. 記憶（Obsidian）

vault: `C:\Users\Owner\OneDrive\Documents\ObsidianVault\株式ダッシュボード\`

- `日次ノート/` — その日の調査ノート（NotebookLMの要点 + Claude Codeへの指示内容を記録）
- `指標/` — 日経平均・為替・Fear&Greed指数・日経平均VI・X投稿分析などの恒久ノート。
  日次ノートからリンクされ、傾向や相関が蓄積されていく
- `Templates/調査ノートテンプレート.md` — 日次ノート作成用テンプレート

調査のたびに日次ノートを作り、関連する指標ノートにリンクを張ることで、
NotebookLMでの調査結果とClaude Codeでの実装判断が指標ごとに関連付けられて蓄積される。
次回の調査時には、指標ノートに溜まった過去の気づきをNotebookLMのソースとして読み込ませることもできる。
